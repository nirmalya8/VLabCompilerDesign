Aim Of The Experiment

To identify whether a given line is a single line comment, multiline comment or neither.



Theory

In programming, comments are hints that a programmer can add to make their code easier to read and understand. Comments are completely ignored by C compilers. There are two ways to add comments in C:
// - Single Line Comment
/*...*/ - Multi-line Comment
In C, a single line comment starts with //. It starts and ends in the same line. In C programming, there is another type of comment that allows us to comment on multiple lines at once, they are multi-line comments. To write multi-line comments, we use the /*....*/ symbol.

Procedure

If the given input line starts with '//' then its a single line comment. If the given line starts with '/' and later ends with '/', then its a multiline comment. Else its not a comment. The process is as follows

  • Take the input
  • Traverse the input line.
  • If two consecutive '/' are found for the first two positions of the input line, then its a single line comment.
  • If '/' is found at the starting and later '/' then it is a multiline comment.
  • if none of the condition satisfies then its not a comment.
